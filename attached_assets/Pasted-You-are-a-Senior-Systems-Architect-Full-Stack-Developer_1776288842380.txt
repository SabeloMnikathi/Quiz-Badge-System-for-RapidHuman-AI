SYSTEM REQUIREMENTS:

1. ADMIN PANEL (/admin)
   - A protected route gated by a hardcoded password checked against an 
     environment variable (VITE_ADMIN_PASSWORD). No user-facing login UI needed.
   - A form to create questions with:
     - Question text (string)
     - 4 radio-button answer options (option A, B, C, D)
     - One correct answer selector
     - Order/position field (so questions appear in the right sequence)
   - Questions are saved to Supabase table "questions" with schema:
     { id uuid PK, text text, options text[], correct_index int2, 
       order int4, created_at timestamptz }
   - An admin list view showing all current questions with edit/delete
   - All Supabase calls use the service-role key on the admin side only

2. USER QUIZ FLOW (/quiz)
   - On load, fetch all questions from Supabase "questions" table 
     ordered by the "order" column ascending
   - Start screen: shows quiz title, a "Begin" button, and explains it is timed
   - Each question fills the entire screen (Typeform-style, one question at a time)
   - Transitions: animate each question sliding in from the right, 
     outgoing slides left — use Framer Motion AnimatePresence for this
   - Users select one radio option, then press "Next" 
     (or auto-advance after selection)
   - Once a question is answered and advanced, the user CANNOT go back 
     (disable/hide back navigation entirely)
   - A countdown timer displays per question (configurable, default 60s per question)
   - If the timer runs out before answering, record a timeout 
     and move to the next question
   - Quiz users are identified via Supabase anonymous sign-in 
     (signInAnonymously via Supabase Auth) — no UI shown to the user
   - The quiz records to Supabase table "attempts":
     { id uuid PK, user_id uuid FK → auth.users, 
       answers jsonb (array of {question_id, selected_index, time_taken}), 
       started_at timestamptz, completed_at timestamptz, 
       passed boolean, badge_url text }
   - Save the attempt row to "attempts" on quiz completion

3. ANSWER GRADING (Supabase Edge Function)
   - A Supabase Edge Function named "grade-attempt" (Deno, TypeScript)
   - HTTP POST trigger, receives { attemptId: string } in the request body
   - Fetches the attempt and all questions from Supabase using the service-role key
   - Compares each answer's selected_index to the question's correct_index
   - If ALL answers are correct:
     — Updates attempt: passed = true
     — Writes to Supabase table "badges":
       { id uuid PK, user_id uuid, awarded_at timestamptz, 
         attempt_id uuid FK, image_url text }
     — image_url points to a badge asset in Supabase Storage bucket "badges"
   - Returns JSON: { passed: boolean, correct_count: number, total: number }
   - Edge Function uses SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY 
     from Deno environment (set in Supabase dashboard secrets)

4. SUCCESS / BADGE SCREEN
   - If passed === true: show a full-screen congratulations view
   - Exact text: "Congratulations, you have unlocked a badge!"
   - Display a badge graphic (PNG from Supabase Storage public URL, 
     or a built-in animated SVG badge as fallback)
   - Add a confetti animation on entry (use canvas-confetti package)
   - If passed === false: show a friendly "Not quite — try again" screen

5. QUIZ RULES (enforced in code, not just UI):
   - No back navigation (block via React Router's useBlocker or 
     window.history.pushState override)
   - Each question is timed (store time_taken per question in the answers jsonb)
   - Only one attempt per browser session (store attemptId in sessionStorage; 
     if present on load, redirect straight to /result)

DATABASE SCHEMA (run this SQL in Supabase SQL Editor):

-- Enable UUID extension
create extension if not exists "uuid-ossp";

-- Questions table
create table questions (
  id uuid primary key default uuid_generate_v4(),
  text text not null,
  options text[] not null,
  correct_index int2 not null,
  "order" int4 not null default 0,
  created_at timestamptz default now()
);

-- Attempts table
create table attempts (
  id uuid primary key default uuid_generate_v4(),
  user_id uuid references auth.users(id),
  answers jsonb not null default '[]',
  started_at timestamptz default now(),
  completed_at timestamptz,
  passed boolean,
  badge_url text
);

-- Badges table
create table badges (
  id uuid primary key default uuid_generate_v4(),
  user_id uuid references auth.users(id),
  attempt_id uuid references attempts(id),
  awarded_at timestamptz default now(),
  image_url text
);

-- Row Level Security
alter table questions enable row level security;
alter table attempts enable row level security;
alter table badges enable row level security;

-- Public read access for questions (quiz users need to fetch them)
create policy "Public can read questions"
  on questions for select using (true);

-- Users can insert and read their own attempts
create policy "Users can insert own attempts"
  on attempts for insert with check (auth.uid() = user_id);

create policy "Users can read own attempts"
  on attempts for select using (auth.uid() = user_id);

-- Users can read their own badges
create policy "Users can read own badges"
  on badges for select using (auth.uid() = user_id);

-- Storage bucket for badge images (set to public in Supabase dashboard)
-- Bucket name: "badges"
